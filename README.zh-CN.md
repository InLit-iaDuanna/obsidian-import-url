# Import URL

[English README](./README.md)

Import URL 是一个 Obsidian 社区插件，可以把公开网页或直链 PDF 导入到你的库中，生成原文笔记，并通过 DeepSeek 风格的模型接口生成结构化知识库笔记。

## 功能简介

- 导入公开文章页面和直链 PDF
- 提取可读正文并生成独立原文笔记
- 生成 AI 结构化 Markdown 笔记
- 生成知识库来源记录和待审核概念候选页；待入库概念默认不写入图谱链接
- 通过知识库管理界面批准、拒绝概念候选页，并控制正式概念是否展示到图谱
- 按首字母、导入时间或链接次数排序概念，并显示每个概念的链接次数
- 重建图谱时只保留已入库概念之间的真实关联，清理单字、数字、hash、未命名和旧链接造成的脏边
- 每次导入前都可以选择模型
- 在库内保留可见的导入历史
- 支持为不同模型单独配置 API 地址
- 默认偏本地使用，可选第三方抓取回退方式，并在设置中明确说明

## 安装

### 手动安装

先构建插件，然后把这些文件复制到：

`<Vault>/.obsidian/plugins/import-url/`

- `manifest.json`
- `main.js`
- `styles.css`

重新加载 Obsidian，然后在 **设置 → 第三方插件 → Import URL** 中启用插件。

## 默认目录结构

插件默认只在 `我的知识库` 下写入文件：

- `我的知识库/成文`：模型整理后的结构化成文笔记。
- `我的知识库/原文`：从网页或 PDF 提取出的原始 Markdown，和成文笔记分开保存。
- `我的知识库/状态/处理中`：导入过程中的临时记录。
- `我的知识库/状态/失败记录`：失败诊断笔记。
- `我的知识库/状态/历史记录`：可见导入记录。
- `我的知识库/概念库/来源`：每次导入对应的来源记录。
- `我的知识库/概念库/待入库`：等待你审核的概念页。
- `我的知识库/概念库/已入库`：批准后的正式概念页。
- `我的知识库/概念库/索引.md`：自动生成的知识库索引。

## 使用方法

1. 打开命令面板，执行 `从 URL 导入`（命令 ID：`import`），或点击侧边栏图标。
2. 粘贴一个公开 URL。
3. 选择模型。
4. 等待插件分别创建原文笔记、AI 整理笔记、知识库来源记录和待入库概念页。
5. （可选）执行 `打开配置文件`（命令 ID：`open-config`）直接编辑库内 `config.toml`。
6. （可选）执行 `打开知识库管理`（命令 ID：`open-wiki-manager`）审核候选概念、切换图谱展示、查看链接次数。
7. （可选）打开候选页后执行 `批准当前知识库候选页` 或 `拒绝当前知识库候选页`。
8. （可选）在知识库管理中点击 `清理旧图谱链接`，移除旧版本 AI 整理笔记里提前生成的概念 wikilink。
9. （可选）在知识库管理中点击 `重建真实关联`，清理插件生成文件里的脏 wikilink，并只在正式概念页之间重建关联。

## 待入库和已入库

- 待入库：模型从导入内容中抽取的待审核结果，先进入 `我的知识库/概念库/待入库`，不会自动覆盖正式知识库，也不会在 AI 整理笔记里提前生成正式概念 wikilink。
- 已入库：批准后的正式知识库页面，默认进入 `我的知识库/概念库/已入库`。你可以在知识库管理界面选择“展示”或“隐藏”；隐藏后不会参与插件重建的真实关联图谱。
- 图谱重建只根据已入库概念页中明确的 `相关概念` 段落生成概念之间的 wikilink，不再因为两个词在同一段正文里同时出现就自动连线。

## 图谱颜色分组

Obsidian 原生图谱的节点颜色通过 **图谱 → 分组** 配置。插件会给生成文件写入稳定标签：

- 概念页：`tag:#import-url/concept`
- 待入库候选：`tag:#import-url/candidate`
- AI 整理成文：`tag:#import-url/article`
- 原文：`tag:#import-url/original`
- 来源/索引/历史/处理中/失败记录：`tag:#import-url/source`、`tag:#import-url/index`、`tag:#import-url/history`、`tag:#import-url/processing`、`tag:#import-url/failed`

建议在图谱分组里给已入库概念、待入库候选、成文、原文分别设置不同颜色；你的手写笔记没有这些标签，会自然保留为默认颜色。插件也会写入 `graph_group` frontmatter，方便你以后按属性搜索或批量筛选。

推荐分组顺序：

1. `tag:#import-url/concept`：概念页，建议用醒目的颜色。
2. `tag:#import-url/candidate`：待入库候选，建议和正式概念页分开。
3. `tag:#import-url/article`：AI 整理成文，建议用较浅颜色。
4. `tag:#import-url/original`：原文，建议用灰色或低饱和颜色。
5. `tag:#import-url/source OR tag:#import-url/index OR tag:#import-url/history OR tag:#import-url/processing OR tag:#import-url/failed`：来源和状态文件，建议用很淡的颜色或在图谱中过滤掉。

你的自有文件不带 `#import-url/...` 标签，因此会作为第三类内容保留默认颜色；如果想单独给手写笔记上色，可以再建一个路径或标签分组。

## 配置项

设置页按 `模型接口`、`模型`、`输出`、`抓取兜底` 分组。

运行时优先级保持不变：

- 已保存的插件设置作为基线
- 导入时读取 `config.toml` 并覆盖对应项

插件支持以下配置：

- 通过 Obsidian 安全存储保存 API 密钥
- 配置 DeepSeek API 地址，默认是 `https://api.deepseek.com`
- 添加额外的自定义模型名称
- 为特定模型单独覆盖 API 地址
- 配置输出、处理中、失败、历史和知识库目录
- 启用可选抓取回退（并在设置中明确披露）

## 平台支持说明

- 核心导入流程兼容桌面和移动端。
- `阅读模式兜底` 会在直接抓取失败或只抓到脚本壳时尝试读取正文；该能力会把来源 URL 发送给 r.jina.ai，默认关闭。
- `浏览器渲染兜底（实验）` 默认关闭，仅限桌面环境，当前仅 macOS。
- 在不支持环境中会清晰失败，不会触发桌面专用 API。

## 常见问题排查

- 提示缺少 API 密钥：
  在设置 `模型接口` → `API 密钥` 保存密钥后再执行导入或连接测试。
- 提示缺少模型：
  在设置 `模型` → `默认模型` 填写模型名称，或在导入弹窗中选择模型。
- `config.toml` 看起来没生效：
  先确认 `模型接口` → `配置文件路径` 路径正确，再重新打开导入弹窗或发起一次新导入触发重载。
- 移动端开启浏览器渲染回退后失败：
  关闭 `浏览器渲染兜底（实验）`；该能力仅限桌面端，当前仅 macOS。
- 感觉导入卡住：
  到你配置的 `历史记录目录` 查看可见历史笔记，确认阶段、进度信息和最近更新时间。

## 发布

发布到 Obsidian 时，请将以下文件作为 GitHub Release 附件上传：

- `manifest.json`
- `main.js`
- `styles.css`

请确保 Git 标签与 `manifest.json` 中的版本号完全一致，且不要添加前导 `v`。

源码事实来源保持在 `src/`；`main.js` 等为构建产物。
